# Troubleshooting

## Issue 1 - Configuration file was overwritten

### Problem

A file lost its previous contents after using output redirection.

### Diagnosis

The `>` operator replaces the destination file if it already exists.

### Solution

Use `>>` when new information should be appended instead of replacing existing content.

---

## Issue 2 - Pipeline returned no results

### Problem

A pipeline completed successfully but displayed no output.

### Diagnosis

The first command in the pipeline produced no matching results.

### Solution

Execute each command separately before combining them into a pipeline to verify the expected output.

---

## Issue 3 - Command output disappeared after scrolling

### Problem

The command generated more output than the terminal window could display.

### Diagnosis

The output scrolled beyond the visible area.

### Solution

Pipe the output to `less` to inspect the information page by page.

Example:

```bash
journalctl | less
```

---

## Issue 4 - Empty file created unexpectedly

### Problem

A new file appeared even though no useful information was written.

### Diagnosis

Output redirection creates the destination file even when the command produces no visible output.

### Solution

Verify the command before redirecting its output.

---

## Issue 5 - Unable to find the expected information

### Problem

The required text could not be located within a large amount of output.

### Diagnosis

Inspecting the complete output manually was inefficient.

### Solution

Filter the output using `grep` before reviewing the results.

Example:

```bash
history | grep git
```

---

## Issue 6 - Incorrect command order in a pipeline

### Problem

The pipeline produced unexpected results.

### Diagnosis

Commands were connected in an incorrect sequence.

### Solution

Remember that each command receives the output generated by the previous one. Test each command individually before combining them.

---

## Issue 7 - Temporary files were created unnecessarily

### Problem

Intermediate files were created only to transfer data between commands.

### Diagnosis

The workflow relied on temporary files instead of command pipelines.

### Solution

Whenever possible, connect commands using pipes to reduce unnecessary file creation and simplify workflows.

---

## Lessons Learned

* Understand the difference between `>` and `>>`.
* Test commands individually before creating pipelines.
* Use pipes instead of temporary files whenever possible.
* Filter information before reading large outputs.
* Save command output only when it is necessary for future analysis.
