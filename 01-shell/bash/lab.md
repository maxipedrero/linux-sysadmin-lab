# Hands-on Lab

## Objective

Practice Bash by solving practical scenarios inspired by OverTheWire Bandit and everyday Linux administration tasks.

**Environment**

* Ubuntu Server (AWS EC2)
* Ubuntu Virtual Machine
* OverTheWire Bandit

---

## Scenario 1 - Reading files with unusual names

### Objective

Access files whose names contain spaces or special characters.

### Commands Used

```bash
ls
cat
pwd
```

### Observation

Learned how Bash interprets spaces and special characters in filenames. Quoting filenames or escaping special characters allows files to be accessed correctly.

---

## Scenario 2 - Exploring an unknown directory

### Objective

Navigate an unfamiliar directory structure and locate specific files.

### Commands Used

```bash
pwd
ls
cd
find
```

### Observation

Practiced filesystem navigation while searching for files and identifying directory structures without relying on graphical tools.

---

## Scenario 3 - Inspecting file contents

### Objective

Read and compare the contents of text files using different commands.

### Commands Used

```bash
cat
less
head
tail
```

### Observation

Learned when each command is more appropriate depending on the file size and the amount of information required.

---

## Scenario 4 - Creating and organizing files

### Objective

Create directories and files directly from the terminal while organizing a project.

### Commands Used

```bash
mkdir
touch
mv
cp
rm
```

### Observation

Practiced common file management operations used during Linux administration and project organization.

---

## Scenario 5 - Building a Git project from Bash

### Objective

Create and manage a Git repository entirely from the command line.

### Commands Used

```bash
git init
git status
git add
git commit
git log
git push
```

### Observation

Managed a complete Git workflow using only Bash, reinforcing confidence with command-line tools.

---

## Skills Practiced

* Filesystem navigation
* File management
* Reading text files
* Searching for files
* Working with Git
* Using the terminal efficiently
* Understanding how Bash interprets filenames
* Solving practical Linux challenges
